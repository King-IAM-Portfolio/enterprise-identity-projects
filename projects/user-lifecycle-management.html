User Lifecycle Management & Identity Administration

Microsoft Entra ID project documenting initial administrator creation, user provisioning, role assignment, license management, and secure user lifecycle operations.

Completed

Executive Summary

A newly provisioned Microsoft Entra ID test tenant required the creation of an initial operational administrator to establish identity management, user provisioning, role administration, and user lifecycle operations before onboarding additional users. This administrator account serves as the foundational identity used to manage Microsoft Entra resources, configure security settings, and perform ongoing directory administration within a non-production environment.

Implementation Workflow

Business Scenario

A fictional organization needed to prepare a new Microsoft Entra ID tenant for identity administration. Before additional users could be onboarded, the tenant required an operational administrator account with appropriate permissions to manage users, roles, licenses, and directory settings.

Project Objectives

  • Create the first operational administrator account in Microsoft Entra ID.
  • Validate the new administrator account in the tenant user list.
  • Assign the Global Administrator role for initial tenant setup activities.
  • Document user lifecycle steps in a clear, recruiter-friendly implementation format.
  • Demonstrate understanding of identity administration, role assignment, and tenant setup.

Environment

  • Platform: Microsoft Entra admin center
  • Tenant Type: Non-production Microsoft Entra ID test tenant
  • Account Type: Cloud-only user account
  • Role Assigned: Global Administrator
  • Purpose: Initial tenant administration and user lifecycle management

Implementation Steps

  1. Navigated to Microsoft Entra admin center and opened the Users blade.
  2. Selected New user to begin creating an operational administrator account.
  3. Entered required identity information including username, display name, and profile properties.
  4. Assigned the Global Administrator role during account creation.
  5. Reviewed the account details and created the user.
  6. Validated that the administrator account appeared successfully in the tenant user list.
  7. Opened the user profile to confirm account details and assigned role configuration.
  8. Verified the tenant user count after the new administrator account was created.

Security Considerations

  • Least Privilege: High-privilege roles should only be assigned when required for tenant setup or administration.
  • Administrative Separation: Administrative accounts should be separate from normal daily-use accounts.
  • Auditability: Administrator creation and role assignment should be documented for review and accountability.
  • Non-Production Scope: This implementation was completed in a test tenant for portfolio and lab documentation purposes.

Skills Demonstrated

Microsoft Entra ID Administration

Created and validated a cloud-only administrator account in Microsoft Entra ID.

Role-Based Access Control

Assigned and verified an administrative directory role for tenant management.

User Lifecycle Management

Documented identity creation, validation, and administrative readiness steps.

Technical Documentation

Created a visual workflow and written an implementation summary for portfolio review.

Lessons Learned

  • Initial tenant setup requires a clear administrative identity strategy.
  • Privileged accounts should be documented and monitored because they can control tenant-wide identity settings.
  • Visual documentation makes identity workflows easier to explain during interviews and technical reviews.
  • Portfolio projects are stronger when they show both the technical steps and the security reasoning behind them.
